## Project Structure

This monorepo contains two independent Astro sites:

```
coderclaw.ai/
├── landing/          # coderclaw.ai — Marketing site (Astro → Cloudflare Workers)
├── docs-site/        # docs.coderclaw.ai — Documentation (Astro Starlight → Cloudflare Pages)
├── scripts/          # Installer CI tests (Docker smoke, matrix, unit)
└── .github/workflows # CI + auto-deploy
```

### `landing/` — Marketing Site

Static Astro site deployed as a Cloudflare Worker with static assets.

| Path | Description |
|------|-------------|
| `src/pages/` | Page routes (index, blog, trust, docs, etc.) |
| `src/data/` | JSON data files (showcase, testimonials) |
| `src/components/` | Reusable Astro components |
| `public/` | Static assets + installer scripts (`install.sh`, `install.ps1`) |
| `wrangler.toml` | Cloudflare Workers config (`[assets]` directive) |

### `docs-site/` — Documentation

Astro Starlight site deployed to Cloudflare Pages.

| Path | Description |
|------|-------------|
| `src/content/docs/` | Documentation pages (MDX/Markdown) |
| `wrangler.toml` | Cloudflare Pages config (`pages_build_output_dir`) |

## Development Setup

### Prerequisites

- Node.js 20+ (required for both sites)
- Bun (optional, used by landing CI — `npm` works locally)
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/install-and-update/) authenticated (`wrangler login`) — only needed for manual deploys

### Install

```sh
# Landing
cd landing && npm install

# Docs
cd docs-site && npm install
```

### Local development

```sh
# Landing (http://localhost:4321)
cd landing && npm run dev

# Docs (http://localhost:4321)
cd docs-site && npm run dev
```

### Build

```sh
# Landing
cd landing && npm run build    # outputs to landing/dist/

# Docs
cd docs-site && npm run build  # outputs to docs-site/dist/
```

## CI/CD — Automatic Deploys

Both sites **auto-deploy on push to `main`** via GitHub Actions, filtered by path:

| Workflow | Trigger | Deploys |
|----------|---------|---------|
| `deploy-landing.yml` | Push to `main` changing `landing/**` | `coderclaw.ai` (Cloudflare Workers) |
| `deploy-docs.yml` | Push to `main` changing `docs-site/**` | `docs.coderclaw.ai` (Cloudflare Pages) |

Both workflows also support `workflow_dispatch` for manual runs from the GitHub Actions UI.

### Required Secrets

The deploy workflows need these GitHub repository secrets:

| Secret | Description |
|--------|-------------|
| `CLOUDFLARE_API_TOKEN` | Cloudflare API token with Workers/Pages deploy permission |
| `CLOUDFLARE_ACCOUNT_ID` | Your Cloudflare account ID |

### Other CI Workflows

| Workflow | Trigger | Purpose |
|----------|---------|---------|
| `install-smoke.yml` | Push to `main`, PRs | Cross-platform installer dry-run tests |
| `install-matrix.yml` | Push to `main`, PRs | Docker matrix tests (Ubuntu, Debian, Fedora) |
| `install-git-smoke.yml` | Daily schedule | Git-based install smoke tests |

## Manual Deploy

If you need to deploy without pushing (e.g., hotfix):

```sh
# Landing (Cloudflare Workers)
cd landing
npm install
npx astro build
npx wrangler deploy

# Docs (Cloudflare Pages)
cd docs-site
npm install
npx astro build
npx wrangler pages deploy dist
```

> **Important:** The landing site is a **Workers** project (`npx wrangler deploy`).
> The docs site is a **Pages** project (`npx wrangler pages deploy dist`).
> Using the wrong command will fail.

1. **Bugs & small fixes** → Open a PR directly.
2. **New features / content** → Open a [GitHub Discussion](https://github.com/SeanHogg/coderclaw.ai/discussions) or ask in Discord first.
3. **Questions** → Discord `#setup-help`.

## Before You PR

- Run the build for any site you changed:
  ```sh
  cd landing && npm run build   # if you touched landing/
  cd docs-site && npm run build # if you touched docs-site/
  ```
- Keep PRs focused: one concern per PR.
- Describe **what** changed and **why**.

## Adding Content

### Blog posts

Add a new `.md` file to `landing/src/content/blog/` with frontmatter matching existing posts.

### Documentation pages

Add or edit `.mdx`/`.md` files in `docs-site/src/content/docs/`. The sidebar is auto-generated by Starlight from the directory structure.

### Showcase / Testimonials

- **Showcase** (`landing/src/data/showcase.json`): grouped by quote length (SHORT / MED / LONG) for uniform CSS grid rows — maintain sort order in the JSON.
- **Testimonials** (`landing/src/data/testimonials.json`): sorted by length, deduplicated by author.
